認識不同的大小寫格式

UPPERCASE 將所有字母轉換為大寫。常用於縮寫、警告、強調及特定格式（如報紙標題或法律文件名稱）。請注意，全大寫文字在大段落中較難閱讀，建議僅用於簡短文字。

lowercase 將所有字母轉換為小寫。適用於文字標準化、為數據處理準備內容（大小寫一致性很重要），或轉換不小心按了 Caps Lock 輸入的文字。

Title Case 將每個單詞的首字母大寫，保留其餘字母的原始大小寫。這是書名、文章標題、章節標題及正式文件標題的標準格式。

camelCase 以小寫字母開頭，後續每個單詞的首字母大寫，無分隔符。這是 JavaScript、TypeScript 等編程語言中變數名稱、函數名稱及物件屬性的標準命名慣例。

PascalCase 每個單詞的首字母大寫，無分隔符（包括首個單詞）。常用於 C# 和 Java 的類別名稱、方法名稱及類型定義。例如 "user profile card" 變成 "UserProfileCard"。

snake_case 用底線分隔單詞，所有字母轉為小寫。這是 Python、Ruby 及許多數據庫系統中變數名稱、函數名稱和欄位名稱的標準慣例。

kebab-case 用連字號分隔單詞，所有字母轉為小寫。這是 URL 字串、CSS 類別名稱及網頁開發檔案名稱最常見的格式。

Capitalize Words 將每個單詞的首字母大寫，其餘字母全部轉為小寫。與 Title Case 不同，此功能會強制一致的大寫格式，不受原文影響。
